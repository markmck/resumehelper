2. **Update `src/main/handlers/export.ts`** to add a themed PDF branch:

   a. Import `renderThemeHtml`, `buildResumeJson` from `../lib/themeRegistry`
   b. Import `profile` from DB schema, `eq` from drizzle-orm (may already be imported), `tmpdir` from `os`, `promises as fs` (already imported)

      ```typescript

      const layoutTemplate = variant?.layoutTemplate ?? 'professional'
      const isProfessional = !layoutTemplate || layoutTemplate === 'professional' || layoutTemplate === 'traditional'
      ```

   e. If `isProfessional`: use the EXISTING code path (load print.html, wait for print:ready). No changes needed.

   f. If NOT professional (theme path):
      - Create hidden BrowserWindow (same size/config as existing, but WITHOUT preload script since we're loading raw HTML):
        ```typescript
        const win = new BrowserWindow({
          show: false,

          height: 1056,
        ```

      - Fetch profile and builder data, build resume.json, render theme HTML:
        ```typescript
        const profileRow = db.select().from(profile).where(eq(profile.id, 1)).get()
        const builderData = await getBuilderDataForVariant(variantId)
        const resumeJson = buildResumeJson(profileRow, builderData)
        const html = await renderThemeHtml(layoutTemplate, resumeJson)
        ```
      - Write HTML to temp file, load it, wait for did-finish-load, add settle delay, then printToPDF:
        ```typescript
        const tmpPath = join(tmpdir(), `resume-theme-${variantId}-${Date.now()}.html`)
        await fs.writeFile(tmpPath, html, 'utf-8')

        await new Promise<void>((resolve) => {
          win.webContents.once('did-finish-load', () => {
            setTimeout(resolve, 500)  // settle delay for fonts/images

          setTimeout(resolve, 5000)  // safety timeout

          printBackground: true,
          pageSize: 'Letter',
          margins: { top: 0, bottom: 0, left: 0, right: 0 },
        })
        win.destroy()
        await fs.unlink(tmpPath).catch(() => {})
        await fs.writeFile(filePath, pdfBuffer)
        return { canceled: false, filePath }
        ```

   b. Add the same `isProfessional` check:
      ```typescript
      const isProfessional = !snapshot.layoutTemplate || snapshot.layoutTemplate === 'professional' || snapshot.layoutTemplate === 'traditional'
      ```

   c. For non-professional snapshots, the challenge is that `themes:renderHtml` takes a `variantId` and fetches live data from DB. But snapshots need to render the FROZEN data, not live data. So we need a different approach:
      - Add a new IPC handler `themes:renderSnapshotHtml` (in themes.ts) that accepts `themeKey` and `snapshotData` (the SubmissionSnapshot object) directly, calls `buildResumeJson(snapshotProfileData, snapshotBuilderData)` using the snapshot's frozen data, then calls `renderThemeHtml(themeKey, resumeJson)`.
      - OR: Create a client-side approach where SnapshotViewer sends the snapshot data to a new IPC channel. The simpler approach is to add a new IPC handler.

   d. **Add to `src/main/handlers/themes.ts`**:
      ```typescript
      ipcMain.handle('themes:renderSnapshotHtml', async (_, themeKey: string, snapshotData: SubmissionSnapshot) => {
        // Build resume.json from snapshot data (frozen data, not live DB)
        // Extract profile from snapshot if available, otherwise use empty profile
        // The snapshot doesn't include profile data directly -- it has jobs, skills, etc.
        // We need to fetch the profile from DB since it's not in the snapshot
        const profileRow = db.select().from(profile).where(eq(profile.id, 1)).get()
        const resumeJson = buildResumeJson(profileRow, {
          jobs: snapshotData.jobs,
          skills: snapshotData.skills,
          projects: snapshotData.projects,
          education: snapshotData.education ?? [],
          volunteer: snapshotData.volunteer ?? [],
          awards: snapshotData.awards ?? [],
          publications: snapshotData.publications ?? [],
          languages: snapshotData.languages ?? [],
          interests: snapshotData.interests ?? [],
          references: snapshotData.references ?? [],
        })
        return renderThemeHtml(themeKey, resumeJson)
      })
      ```

   e. **Add to preload bridge** (`src/preload/index.ts` themes section):
      ```typescript
      renderSnapshotHtml: (themeKey: string, snapshotData: unknown) =>
        ipcRenderer.invoke('themes:renderSnapshotHtml', themeKey, snapshotData),
      ```

   f. **Add to preload types** (`src/preload/index.d.ts` Api.themes):
      ```typescript
      renderSnapshotHtml: (themeKey: string, snapshotData: SubmissionSnapshot) => Promise<string | { error: string }>
      ```

   g. **In SnapshotViewer.tsx**:
      - Add state: `const [themeHtml, setThemeHtml] = useState<string | null>(null)` and `const [loading, setLoading] = useState(!isProfessional)`
      - Add useEffect: if `!isProfessional`, call `window.api.themes.renderSnapshotHtml(snapshot.layoutTemplate, snapshot).then(setThemeHtml).finally(() => setLoading(false))`
      - In render: if `isProfessional`, render ProfessionalLayout as before. If loading, show loading indicator inside the modal body. If `themeHtml`, render an iframe with `srcDoc={themeHtml}` inside the modal body:
