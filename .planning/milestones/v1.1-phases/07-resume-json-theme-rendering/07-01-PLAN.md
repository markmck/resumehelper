1. **Create `src/main/lib/themeRegistry.ts`** with:

   a. `ThemeEntry` interface (`key: string`, `displayName: string`)

   b. `THEMES` array:
      - `{ key: 'professional', displayName: 'Professional (built-in)' }`
      - `{ key: 'even', displayName: 'Even' }`
      - `{ key: 'class', displayName: 'Class' }`
      - `{ key: 'elegant', displayName: 'Elegant' }`

   c. `THEME_KEYS` derived from THEMES

   d. `buildResumeJson(profileRow, builderData)` function that maps BuilderData to resume.json schema:
      - Profile -> `basics` (name, email, phone, location as `{ city }`, linkedin as profiles array entry)
      - Jobs (non-excluded) -> `work` array: `company` -> `name`, `role` -> `position`, bullets (non-excluded) -> `highlights`
      - Skills (non-excluded) grouped by first tag -> `skills` array: tag -> `name`, skill names -> `keywords`. Skills with no tags go under "Other" group.
      - Projects (non-excluded) -> `projects` array: bullets (non-excluded) -> `highlights`
      - Education, volunteer, awards, publications, languages, interests, references all mapped per resume.json schema (see RESEARCH.md buildResumeJson example for exact field mappings)
      - Import types from `../../preload/index.d` for BuilderData and Profile

   e. `renderThemeHtml(themeKey, resumeJson)` async function:
      - `'even'`: `const theme = await import('jsonresume-theme-even'); return theme.render(resumeJson)`
      - `'class'`: `const theme = await import('@jsonresume/jsonresume-theme-class'); return theme.render(resumeJson)`
      - `'elegant'`: use `require('jsonresume-theme-elegant')` (CJS), call `theme.render(resumeJson)`, wrap in `Promise.resolve()` since it may be sync
      - Default: throw `Error('Unknown theme: ...')`

2. **Create `src/main/handlers/themes.ts`** with `registerThemeHandlers()`:
   - `ipcMain.handle('themes:renderHtml', async (_, variantId: number, themeKey: string) => { ... })`
   - Inside: fetch profile from DB (`db.select().from(profile).where(eq(profile.id, 1)).get()`), get builder data using `getBuilderDataForVariant(variantId)` (import from `./export.ts` -- NOTE: `getBuilderDataForVariant` is not currently exported from export.ts, so first export it from export.ts), call `buildResumeJson(profileRow, builderData)`, call `renderThemeHtml(themeKey, resumeJson)`, return the HTML string
   - Wrap in try/catch: on error, return `{ error: err.message }` so renderer can show fallback
   - Also add `ipcMain.handle('themes:list', () => THEMES)` so the renderer can get the theme list dynamically

3. **Update `src/main/handlers/export.ts`**: Add `export` keyword to the `getBuilderDataForVariant` function so themes.ts can import it: `export async function getBuilderDataForVariant(...)`.

4. **Update `src/main/handlers/index.ts`**: Import and call `registerThemeHandlers()`.

5. **Update `src/preload/index.ts`**: Add `themes` namespace to the `api` object:
   ```typescript
   themes: {
     renderHtml: (variantId: number, themeKey: string) =>
       ipcRenderer.invoke('themes:renderHtml', variantId, themeKey),
     list: () => ipcRenderer.invoke('themes:list'),
   },
   ```

6. **Update `src/preload/index.d.ts`**: Add themes to the `Api` interface:
   ```typescript
   themes: {
     renderHtml: (variantId: number, themeKey: string) => Promise<string | { error: string }>
     list: () => Promise<Array<{ key: string; displayName: string }>>
   }
   ```

7. **Update `src/renderer/src/components/VariantEditor.tsx`**:
   - Add state: `const [themes, setThemes] = useState<Array<{key: string; displayName: string}>>([])` and `const [layoutTemplate, setLayoutTemplate] = useState(variant.layoutTemplate || 'professional')`
   - Add useEffect to fetch themes on mount: `window.api.themes.list().then(setThemes)`
   - Treat both `'traditional'` and `'professional'` as the built-in layout (the DB default is 'traditional' from earlier phases)
   - Add a theme dropdown in the Preview sub-tab header area (next to export buttons), styled as a `<select>` with dark theme classes consistent with existing button styles:
     ```
     <select value={layoutTemplate} onChange={handleThemeChange} className="...dark theme styles...">
       {themes.map(t => <option key={t.key} value={t.key}>{t.displayName}</option>)}
     </select>
     ```
   - `handleThemeChange`: call `window.api.templates.setLayoutTemplate(variant.id, newTheme)` then `setLayoutTemplate(newTheme)`. The setLayoutTemplate IPC already exists and is wired.
   - Pass `layoutTemplate` as a new prop to VariantPreview: `<VariantPreview variantId={variant.id} layoutTemplate={layoutTemplate} />`
   - When `variant` prop changes (different variant selected), sync `layoutTemplate` state: add `useEffect` that sets `setLayoutTemplate(variant.layoutTemplate || 'professional')` when `variant.id` changes. Treat 'traditional' as 'professional'.
