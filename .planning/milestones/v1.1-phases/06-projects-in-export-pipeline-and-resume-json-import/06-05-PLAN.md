<objective>
Implement resume.json file import with a two-step flow: parse and show confirmation, then atomically replace all data in a synchronous transaction.

Purpose: Users with existing resume.json files (from LinkedIn exports, other tools, or manual creation) can import all their data at once instead of manually entering each item.

Output: "Import from resume.json" button on Experience tab, native file picker, confirmation modal with counts, atomic data replacement, success toast.
</objective>

  <action>
1. Create `src/main/handlers/import.ts` with `registerImportHandlers()`:

   **`import:parseResumeJson` handler (async):**
   - Call `dialog.showOpenDialog({ title: 'Import resume.json', filters: [{ name: 'JSON Files', extensions: ['json'] }], properties: ['openFile'] })`
   - If canceled, return `{ canceled: true }`
   - Read file with `fs.promises.readFile(filePath, 'utf-8')`
   - `JSON.parse(raw)` wrapped in try/catch — on error return `{ canceled: false, error: 'Invalid JSON file' }`
   - Count sections: `{ jobs: (data.work ?? []).length, skills: (data.skills ?? []).length, projects: (data.projects ?? []).length, education: (data.education ?? []).length, volunteer: (data.volunteer ?? []).length, awards: (data.awards ?? []).length, publications: (data.publications ?? []).length, languages: (data.languages ?? []).length, interests: (data.interests ?? []).length, references: (data.references ?? []).length, hasProfile: !!data.basics }`
   - Return `{ canceled: false, counts, data }` — pass parsed data to renderer for round-trip on confirm

   **`import:confirmReplace` handler (sync transaction):**
   - Receives the parsed resume.json data object from renderer
   - Uses `sqlite.transaction()` (import `sqlite` — need to export it from db/index.ts, or import Database instance directly). CRITICAL: The sqlite instance is created in db/index.ts. Export it: `export { sqlite }` from db/index.ts (or use `db` with Drizzle — but the transaction MUST be synchronous, so use raw sqlite).
   - Actually, check if `sqlite` is already exported. If not, add `export { sqlite }` to `src/main/db/index.ts`.
   - Transaction body (ALL synchronous — no await):
     a. Delete in child-first order to avoid FK issues:
        ```
        sqlite.prepare('DELETE FROM template_variant_items').run()
        sqlite.prepare('DELETE FROM job_bullets').run()
        sqlite.prepare('DELETE FROM jobs').run()
        sqlite.prepare('DELETE FROM skills').run()
        sqlite.prepare('DELETE FROM project_bullets').run()
        sqlite.prepare('DELETE FROM projects').run()
        sqlite.prepare('DELETE FROM education').run()
        sqlite.prepare('DELETE FROM volunteer').run()
        sqlite.prepare('DELETE FROM awards').run()
        sqlite.prepare('DELETE FROM publications').run()
        sqlite.prepare('DELETE FROM languages').run()
        sqlite.prepare('DELETE FROM interests').run()
        sqlite.prepare('DELETE FROM "references"').run()
        ```
        NOTE: `references` may need quoting if it's a reserved word in SQLite — use double quotes.

     b. Import profile (UPDATE, not INSERT — row already exists from ensureSchema):
        ```
        if (parsed.basics) {
          sqlite.prepare('UPDATE profile SET name=?, email=?, phone=?, location=?, linkedin=? WHERE id=1')
            .run(
              parsed.basics.name ?? '',
              parsed.basics.email ?? '',
              parsed.basics.phone ?? '',
              parsed.basics.location?.city ?? '',
              (parsed.basics.profiles?.[0]?.url ?? '')
            )
        }
        ```

     c. Import work -> jobs + job_bullets:
        For each `parsed.work[]`: INSERT into jobs (company=work.name, role=work.position, start_date truncated to YYYY-MM, end_date truncated or null). Get lastInsertRowid. For each highlight, INSERT into job_bullets with sort_order.

     d. Import skills -> skills:
        For each `parsed.skills[]`: INSERT into skills (name=skill.name, tags=JSON.stringify(skill.keywords ?? []))

     e. Import projects -> projects + project_bullets:
        For each `parsed.projects[]`: INSERT into projects (name=project.name, sort_order incrementing). Get lastInsertRowid. For each highlight, INSERT into project_bullets with sort_order.

     f. Import education -> education:
        For each `parsed.education[]`: INSERT with institution, area, studyType (field: studyType), startDate (truncated), endDate (truncated), score, courses as JSON.stringify(education.courses ?? [])

     g. Import volunteer -> volunteer:
        For each `parsed.volunteer[]`: INSERT with organization, position, startDate, endDate, summary, highlights as JSON.stringify(volunteer.highlights ?? [])

     h. Import awards, publications, languages, interests, references:
        Each maps directly per the field mapping in RESEARCH.md. interests.keywords and similar JSON array fields get JSON.stringify'd.

     i. Date truncation helper: `const truncDate = (d) => d ? d.slice(0, 7) : null` — applied to all date fields

   - Return `{ success: true }` or throw on error (transaction auto-rolls back)

2. In `src/main/db/index.ts`: Add `export { sqlite }` so the import handler can access the raw Database instance for synchronous transactions.

     ```typescript
     const result = await window.api.import_.parse()
     if (result.canceled) return
     if (result.error) { showToast(result.error); return }
     ```
