<task type="auto">
  <name>Task 2: Unify VariantPreview to iframe-only path with zoom-to-fit</name>
  <files>src/renderer/src/components/VariantPreview.tsx</files>
  <action>
Replace the entire VariantPreview component with a unified iframe-only implementation. Remove:
- `isBuiltIn()` function
- `themeHtml` state
- `themeLoading` state
- `builderData` state
- `profileData` state
- All direct-render of ProfessionalLayout
- The `srcDoc` iframe path for themes
- Import of ProfessionalLayout
- Imports of BuilderData, Profile from preload

New implementation:
1. Props interface stays the same: `{ variantId, layoutTemplate?, refreshKey? }`
2. Use `useRef<HTMLDivElement>` for the container to measure width
3. Use `useState<number>` for scale factor
4. Use `useEffect` + `ResizeObserver` on container to compute `scale = containerWidth / 816` on mount and resize
5. Build print URL: `const base = (window as Window & { __printBase?: string }).__printBase ?? window.location.origin; const printUrl = \`${base}/print.html?variantId=${variantId}&template=${layoutTemplate ?? 'classic'}\``
6. Render structure per CONTEXT.md locked decisions:

```tsx
<div ref={containerRef} style={{
  background: 'var(--color-surface-raised)',  // dark gray, PDF-viewer feel
  overflowY: 'auto',
  height: '100%',
  padding: '16px',
}}>
  <div style={{
    width: '100%',
    height: `${1056 * scale}px`,
    overflow: 'hidden',
  }}>
    <iframe
      key={`${variantId}-${layoutTemplate}-${refreshKey}`}
      src={printUrl}
      style={{
        width: '816px',
        height: '1056px',
        border: 'none',
        transformOrigin: 'top left',
        transform: `scale(${scale})`,
        display: 'block',
      }}
    />
  </div>
</div>
```

7. The `key` prop forces iframe reload when variantId, layoutTemplate, or refreshKey changes — this IS the auto-refresh mechanism (no separate debounce needed at this level; VariantEditor already debounces refreshKey increments)
8. Loading state: show "Loading..." while scale is 0 (before first measurement)
9. Gray background behind white paper = `var(--color-surface-raised)` from design system tokens

**Important per CONTEXT.md:** Visible page gaps (gray strip between page 1 and page 2) are foundation for Phase 15. For now, the iframe is 1056px (one page). Multi-page content will scroll within the iframe. Phase 15 adds the page boundary visualization. Do NOT attempt multi-page gap rendering in this task.

**Verify:** `var(--color-surface-raised)` exists in the design system. Check `src/renderer/src/styles/` or similar for this token. If the exact token name differs, use the actual dark gray surface token from the codebase.
