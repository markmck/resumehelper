**Font bundling:**
1. Create `src/renderer/public/fonts/` directory
2. Download woff2 files from Google Fonts CDN. Use the Google Fonts API CSS2 URL pattern to get direct woff2 URLs:
   - Inter: weights 400, 700
   - Lato: weights 300, 400, 700
   - EB Garamond: weight 400 normal, weight 400 italic
3. Save as: `inter-regular.woff2`, `inter-bold.woff2`, `lato-light.woff2`, `lato-regular.woff2`, `lato-bold.woff2`, `eb-garamond-regular.woff2`, `eb-garamond-italic.woff2`

To download, use curl with the Google Fonts CSS2 API. Add a User-Agent header that requests woff2:
```
curl -s "https://fonts.googleapis.com/css2?family=Inter:wght@400;700&display=swap" -H "User-Agent: Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36"
```
Parse the returned CSS for the woff2 URLs, then download each file individually. If Google Fonts is unreliable, use fontsource npm packages as an alternative source (e.g., `@fontsource/inter` contains woff2 files).

<task type="auto">
  <name>Task 2: Expose __printBase preload global for iframe URL construction</name>
  <files>src/preload/index.ts</files>
  <action>
Add a `__printBase` global to the preload's contextBridge so the renderer can build correct print.html URLs in both dev and prod.

In `src/preload/index.ts`, before the existing `contextBridge.exposeInMainWorld('api', api)` call, add:

```typescript
// Print URL base for iframe src in both dev and prod
const printBase = process.env['ELECTRON_RENDERER_URL']
  ?? (() => {
    // In prod, __dirname points to the preload output dir.
    // The renderer's print.html is at ../renderer/print.html relative to preload.
    // For file:// iframe src, we need the renderer directory as a file:// URL.
    const path = require('path')
    const rendererDir = path.join(__dirname, '../renderer')
    return 'file://' + rendererDir.replace(/\\/g, '/')
  })()

contextBridge.exposeInMainWorld('__printBase', printBase)
```

**Important:** Test this approach works. If `import.meta.url` is available in the preload context, prefer:
```typescript
const printBase = process.env['ELECTRON_RENDERER_URL']
  ?? new URL('../renderer', 'file://' + __dirname.replace(/\\/g, '/')).href
```

The key requirement: in dev, `__printBase` = `http://localhost:5173`. In prod, `__printBase` = `file:///C:/path/to/app/renderer` (the directory containing print.html).

Also add TypeScript declaration so renderer code can access it without errors. Add to `src/preload/index.d.ts`:
```typescript
declare global {
  interface Window {
    __printBase?: string
  }
}
```
Or if the declaration pattern in the project uses a different approach, follow the existing pattern.

**Spike note (Claude's discretion):** If the preload global approach proves problematic during testing, fall back to relative `./print.html` path. The executor should test by running `npm run dev` and checking `console.log(window.__printBase)` in the renderer devtools.
