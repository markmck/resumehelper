**ClassicTemplate.tsx:** 1:1 visual port of ProfessionalLayout.tsx with these specific differences per CONTEXT.md locked decisions:

1. Import `ResumeTemplateProps` from `./types` and `filterResumeData` from `./filterResumeData`
2. Component signature: `export default function ClassicTemplate({ profile, accentColor = '#cccccc', ...props }: ResumeTemplateProps)`
3. Call `filterResumeData({ profile, accentColor, ...props })` at top of component
4. **Font:** `fontFamily: "'Times New Roman', 'Times', serif"` (ProfessionalLayout uses Calibri/Segoe UI — this IS a change)
5. **Section headings:** UPPERCASE via `textTransform: 'uppercase'` (ProfessionalLayout already does this — keep it)
6. **accentColor:** Use in `borderBottom: \`1px solid ${accentColor}\`` on sectionHeadingStyle (ProfessionalLayout hardcodes #cccccc)
7. **Summary:** If `profile?.summary` exists, render as a plain `<div>` with paragraph styling directly below the contact info line. NO `<h2>Summary</h2>` heading above it. If summary is empty/undefined, skip entirely — no empty space.
8. All styles are inline React CSSProperties (no CSS files, no Tailwind)
9. Outer wrapper: `maxWidth: '8.5in'`, `margin: '0 auto'`, `padding: '0.5in'`, `backgroundColor: '#ffffff'`
10. Port ALL sections from ProfessionalLayout: header/contact, summary (modified), work experience, skills (grouped by tag), projects, education, volunteer, awards, publications, languages, interests, references
11. For each section: only render if there are included items (e.g., `includedVolunteer.length > 0`)
12. Job bullets: render only non-excluded bullets from filterResumeData output

Read ProfessionalLayout.tsx fully to port every section and style. The goal is visual parity (except font and summary heading changes).
