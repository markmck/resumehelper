to:

The iframe viewer never has Chromium page margins, so the PDF export should not either. The normal PDF export (export:pdf handler at line ~279) already has the correct behavior since template CSS and printToPDF margins are coordinated there — but for snapshots, the template CSS padding already provides all spacing, so Chromium margins must be zero.

Also remove the now-unused `marginDefaults` variable (line ~836) since it's no longer referenced. If it IS referenced elsewhere in the handler (e.g., in the postMessage data), keep it but just zero the printToPDF call.
