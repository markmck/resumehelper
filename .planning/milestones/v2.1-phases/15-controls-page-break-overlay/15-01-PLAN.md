  <files>src/main/db/schema.ts, src/main/db/index.ts, src/main/handlers/templates.ts</files>
  <action>
1. **schema.ts** — Add `templateOptions: text('template_options')` to the `templateVariants` table definition (after `layoutTemplate`, before `createdAt`).

2. **db/index.ts** — Append to the `alterStatements` array:
   `'ALTER TABLE \`template_variants\` ADD COLUMN \`template_options\` text'`

3. **templates.ts** — Add two new IPC handlers:

   `templates:getOptions(variantId: number)` — SELECT the variant row, parse `templateOptions` as JSON, return the parsed object or null if column is null/empty. Use try/catch around JSON.parse.

   `templates:setOptions(variantId: number, options: object)` — UPDATE the row, set `templateOptions` to `JSON.stringify(options)`. The options object shape: `{ accentColor?: string, skillsDisplay?: 'grouped' | 'inline', marginTop?: number, marginBottom?: number, marginSides?: number }`.

4. **templates.ts** — Modify `templates:list` handler to include `templateOptions` in the SELECT return. The returned TemplateVariant objects should include `templateOptions` as a parsed JSON object (or null).

5. **templates.ts** — Add `else if (itemType === 'summary')` branch to `templates:setItemExcluded`:
   - Delete any existing row where `variantId` matches AND `itemType === 'summary'`
   - If `excluded` is true, insert new row with `{ variantId, itemType: 'summary', excluded: true }`
   - Use `itemId` parameter but ignore its value (summary has no real row ID; the sentinel 0 is passed by the renderer)

6. **templates.ts** — Modify `templates:create` handler: after inserting the new variant, check `data.layoutTemplate ?? 'classic'`. If the template is NOT 'executive', insert a `template_variant_items` row with `{ variantId: newRow.id, itemType: 'summary', excluded: true }` to default summary to hidden. Executive templates skip this (summary shown by default = no exclusion row). Note: `templates:create` currently only accepts `{ name }` — also accept optional `layoutTemplate` in the data parameter so the caller can pass the template choice.

7. **templates.ts** — Modify `templates:getBuilderData` to include `summaryExcluded: boolean` in the response. Query `template_variant_items` for a row where `variantId` matches AND `itemType === 'summary'` AND `excluded === true`. If found, `summaryExcluded = true`; else `summaryExcluded = false`.
  </action>
