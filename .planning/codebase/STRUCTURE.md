## Directory Purposes

**`src/main/`:**
- Purpose: Electron main process - handles OS operations, window management, IPC, and database
- Contains: Lifecycle handlers, window creation, IPC setup, all Node.js code
- Key files: `src/main/index.ts` (entry point)

**`src/main/db/`:**
- Purpose: Data persistence layer using Drizzle ORM with SQLite backend
- Contains: Database connection, schema definitions, migrations
- Key files: `src/main/db/index.ts` (singleton db export), `src/main/db/schema.ts` (table definitions)

**`src/preload/`:**
- Purpose: Security bridge between main and renderer processes
- Contains: Context bridge setup, Electron API exposure, type definitions
- Key files: `src/preload/index.ts` (bridge setup), `src/preload/index.d.ts` (global types)

**`src/renderer/src/`:**
- Purpose: React user interface application
- Contains: React components, styling, assets, entry point
- Key files: `src/renderer/src/main.tsx` (React root), `src/renderer/src/App.tsx` (root component)

**`src/renderer/src/components/`:**
- Purpose: Reusable React components
- Contains: Component implementations with local state/logic
- Key files: `src/renderer/src/components/Versions.tsx` (displays Electron/Chrome/Node versions)

**`src/renderer/src/assets/`:**
- Purpose: Static resources for UI
- Contains: CSS stylesheets, SVG images, icons
- Key files: `src/renderer/src/assets/main.css`, `src/renderer/src/assets/base.css`

**`resources/`:**
- Purpose: Application-level resources included in distribution
- Contains: Icon files, installer assets
- Key files: `resources/icon.png` (used on Linux builds)

**`drizzle/`:**
- Purpose: Generated SQL migrations and schema history
- Contains: Migration files (auto-generated by drizzle-kit)
- Generated: Yes
- Committed: Yes (to track schema evolution)

**`build/`:**
- Purpose: Electron builder staging directory
- Contains: Packaged application files
- Generated: Yes
- Committed: No

**`out/`:**
- Purpose: Compiled TypeScript output before Electron packaging
- Contains: Main.js, preload.js, renderer assets
- Generated: Yes
- Committed: No

**Testing:**

## Naming Conventions

**Files:**
- **TypeScript files**: camelCase with .ts extension (e.g., `main.ts`, `schema.ts`)
- **React components**: PascalCase with .tsx extension (e.g., `App.tsx`, `Versions.tsx`)
- **CSS files**: kebab-case with .css extension (e.g., `main.css`, `base.css`)
- **Config files**: Specific naming per tool (e.g., `drizzle.config.ts`, `.prettierrc.yaml`)

**Directories:**
- **Source directories**: lowercase (e.g., `src`, `main`, `renderer`, `preload`, `db`, `components`, `assets`)
- **Build output**: specific names (e.g., `out`, `build`, `drizzle`)

**TypeScript/JavaScript Naming:**
- **Functions**: camelCase (e.g., `createWindow()`, `ipcHandle()`)
- **Variables**: camelCase (e.g., `mainWindow`, `dbPath`, `versions`)
- **Types/Interfaces**: PascalCase (e.g., `ElectronAPI`)
- **Database tables**: lowercase with underscores (e.g., `users` table with `created_at` column)

## Where to Add New Code

**New Feature (e.g., user management):**
- Backend/database logic: `src/main/` - create new handler or extend `src/main/db/schema.ts` for new tables
- IPC handler: `src/main/index.ts` - add `ipcMain.handle()` or `ipcMain.on()` call
- UI component: `src/renderer/src/components/` - create new .tsx file
- Tests: Not applicable (no test infrastructure present)

**New Database Table:**
- Implementation: `src/main/db/schema.ts` - define new table with Drizzle
- Queries: `src/main/db/index.ts` or new file in `src/main/db/` - export query functions
- Migrations: Auto-generated by drizzle-kit (run `drizzle-kit generate` after schema changes)

**New React Component:**
- Implementation: `src/renderer/src/components/ComponentName.tsx` - functional component with hooks
- Usage: Import in parent component (e.g., `src/renderer/src/App.tsx`)
- Styling: Inline Tailwind classes or link to CSS in `src/renderer/src/assets/`

**New IPC Communication:**
- Main handler: `src/main/index.ts` - add `ipcMain.handle('channel-name', ...)` or `ipcMain.on(...)`
- Preload exposure: `src/preload/index.ts` - optionally add to `api` object if custom
- Renderer call: Use `window.electron.ipcRenderer.send/invoke('channel-name', ...)`

**Utilities/Helpers:**
- Main process utilities: `src/main/utils/` (new directory) - shared logic for handlers
- Renderer utilities: `src/renderer/src/utils/` (new directory) - shared component logic
