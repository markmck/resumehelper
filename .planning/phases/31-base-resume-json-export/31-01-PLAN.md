    **(a) src/main/lib/settings.ts** — create new file. Pure helpers, handler-extraction style (db: Db first arg, no IPC/dialog/fs):
    ```ts
    import { eq } from 'drizzle-orm'
    import type { BetterSQLite3Database } from 'drizzle-orm/better-sqlite3'
    import type * as schema from '../db/schema'
    import { appSettings } from '../db/schema'

    type Db = BetterSQLite3Database<typeof schema>

    /**
     * Read a string-valued setting from the app_settings k/v table.
     * Returns undefined when the key is absent.
     */
    export function getSetting(db: Db, key: string): string | undefined {
      const row = db.select().from(appSettings).where(eq(appSettings.key, key)).get()
      return row?.value
    }

    /**
     * Write/overwrite a string-valued setting (UPSERT — no UNIQUE collision).
     */
    export function setSetting(db: Db, key: string, value: string): void {
      db.insert(appSettings)
        .values({ key, value })
        .onConflictDoUpdate({ target: appSettings.key, set: { value } })
        .run()
      }
    ```

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-31-01 | Tampering | `app_settings` k/v table | accept | Single-user local SQLite DB; if an attacker has DB write access they already own the data layer. No integrity verification at the row level. |
| T-31-02 | Information Disclosure | `lastExportDir` stored in `app_settings` | accept | Filesystem path is not sensitive (the user picked it). Stored unencrypted at rest — consistent with all other app data. |
| T-31-03 | Injection | `setSetting(db, key, value)` | mitigate | Drizzle parameterizes the INSERT/ON CONFLICT statement — `key`/`value` cannot break out of the prepared statement. Verified by Drizzle's `eq()` + `.values({key, value})` API contract. No raw SQL concatenation. |
| T-31-04 | Denial of Service | Unbounded growth of `app_settings` rows | accept | Single-user app; k/v table has < 10 keys for the foreseeable future. No mitigation needed at this scale. |
| T-31-05 | Path-shape leak via `value` field | `lastExportDir` | accept | Plan 03 writes `path.dirname(filePath)` — already trusted source (Electron save dialog returns an OS-validated path). No path-traversal vector originates from this plan. |

No `high` severity threats. All threats are `accept` or `mitigate` with Drizzle parameterization as the active mitigation.
