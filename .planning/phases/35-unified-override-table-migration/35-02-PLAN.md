## Trust Boundaries

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-35-03 | Tampering / Data Integrity | INSERT-SELECT migration | mitigate | All-or-nothing `sqlite.transaction()` (D-05) — a partial failure rolls back, leaving entity_overrides empty and analysis_bullet_overrides intact. No partial/corrupt state. |
| T-35-04 | Information Disclosure / Injection | migration + assertion SQL | mitigate | SQL is static/parameterized via better-sqlite3 prepare; no string-concatenated user input reaches the query (no SQL-injection surface). |
| T-35-05 | Denial of Service | startup assertion | mitigate | Assertion warns-but-allows (D-05) — a count mismatch never hard-blocks app launch, so a migration edge case cannot brick the app. |
| T-35-06 | Repudiation / silent data loss | NULL-variant rows | mitigate | NULL-variant rows are explicitly counted and logged (skip-with-warn); the assertion accounts for them so a legitimate skip is not mistaken for loss and a real loss is surfaced. |
| T-35-SC | Tampering | npm installs | mitigate | This plan adds NO new packages. If an install is attempted, stop and route to a blocking-human legitimacy checkpoint. |
</threat_model>
