<decisions_in_effect>
- D-01: per-entity nullable FK columns (bullet_id, project_id, job_id, project_bullet_id), each `onDelete: 'cascade'`, plus `entity_type` + `field` discriminators. NOT a generic entity_id.
- D-02: summary entity carries NO FK (all FK columns NULL, entity_type='summary', field='text'). Sentinel/NULL, not a fake FK.
- D-03: scope columns variant_id (ŌåÆ template_variants, cascade) and analysis_id (ŌåÆ analysis_results, cascade), BOTH nullable. NULL semantics encode the tier.
- D-04: provision all four FK columns now (bullet_id, project_id, job_id, project_bullet_id) ŌĆö forward-looking column set, zero cost until used. This is NOT a green light to implement job_title/company/project reword (out of scope v2.6).
- D-07: keep analysis_bullet_overrides defined and created (read-only). Do NOT drop it.
- RESOLVED ŌĆö Uniqueness enforcement: option (a) partial unique indexes (NOT app-level only). Two indexes ŌĆö one `WHERE analysis_id IS NULL` (variant-tier), one `WHERE analysis_id IS NOT NULL` (analysis-tier) ŌĆö each keyed on scope column + entity_type + all four FK columns + field. SQLite treats NULLs as distinct in plain UNIQUE constraints, so partial indexes are required to dedupe per tier.
</decisions_in_effect>
