  <files>src/main/db/schema.ts, src/main/handlers/submissions.ts, src/main/handlers/index.ts</files>
  <action>
1. **Schema update** (`src/main/db/schema.ts`): Add `url: text('url')` and `notes: text('notes')` columns to the `submissions` table definition (both nullable, no `.notNull()`). Place them after `resumeSnapshot`.

2. **Generate migration**: Run `npx drizzle-kit generate` to create the migration SQL file under `drizzle/`. Verify the generated file contains two ALTER TABLE ADD COLUMN statements. Commit the generated migration file alongside schema.ts.

3. **Create submissions handler** (`src/main/handlers/submissions.ts`): Export `registerSubmissionHandlers()` with four IPC handlers:

   - `submissions:list` — Query submissions table with LEFT JOIN on `templateVariants` to get variant name. Return rows ordered by `desc(submissions.submittedAt)`. Each returned row should include all submission columns plus `variantName: string | null` (null when variant was deleted). Use Drizzle's `leftJoin` API: `db.select({ ...columns }).from(submissions).leftJoin(templateVariants, eq(submissions.variantId, templateVariants.id)).orderBy(desc(submissions.submittedAt))`. Map the result to flatten the joined shape into a clean object.

   - `submissions:create` — Accepts `{ company, role, submittedAt, variantId, url?, notes? }`. If `variantId` is not null, call `buildSnapshotForVariant(variantId)` to capture the frozen snapshot. The snapshot shape is `{ layoutTemplate: string, jobs: BuilderJob[], skills: BuilderSkill[] }` — fetch the variant's `layoutTemplate` from `templateVariants` table alongside the builder data. Store `JSON.stringify(snapshot)` in `resumeSnapshot`. Default `submittedAt` to `new Date()` if not provided. Insert and return the new row (with variantName from a follow-up select, or just return the inserted row — the list will re-fetch anyway).

   - `submissions:update` — Accepts `(id, { company?, role?, submittedAt?, url?, notes? })`. Intentionally does NOT accept `resumeSnapshot` — the snapshot is immutable. Update and return.

   - `submissions:delete` — Accepts `(id)`. Delete the row.

   **`buildSnapshotForVariant` helper** (private function in same file): Replicate the query logic from `templates:getBuilderData` in `src/main/handlers/templates.ts` (lines 76-128). Additionally fetch the variant's `layoutTemplate` from `templateVariants`. Return `{ layoutTemplate, jobs, skills }`. Import all needed schema tables: `submissions, templateVariants, templateVariantItems, jobs, jobBullets, skills`.

   Note: Do not extract a shared utility from templates.ts — duplication here is acceptable since the snapshot version adds `layoutTemplate` and may diverge from the live builder query in the future.

4. **Register handler** (`src/main/handlers/index.ts`): Import `registerSubmissionHandlers` from `'./submissions'` and call it inside `registerAllHandlers()`.
