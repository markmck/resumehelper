**SubmissionAddForm.tsx** — inline form component (follows JobAddForm pattern):
- State: `company` (string, required), `role` (string, required), `submittedAt` (string, defaults to `new Date().toISOString().split('T')[0]`), `variantId` (number | null, default null), `url` (string, optional), `notes` (string, optional).
- On mount: fetch variants via `window.api.templates.list()` and store in local state for the dropdown.
- Form fields (use inline styles for vertical spacing — `style={{ marginBottom: '...' }}` — Tailwind v4 spacing utilities are unreliable in this project):
  - Company: `<input type="text">` (required)
  - Role: `<input type="text">` (required)
  - Date: `<input type="date">` defaulting to today (YYYY-MM-DD)
  - Variant: `<select>` with `<option value="">-- No variant --</option>` plus one option per variant
  - URL: `<input type="url">` (optional)
  - Notes: `<textarea rows={2}>` (optional)
  - Save + Cancel buttons
- On save: call `window.api.submissions.create({ company, role, submittedAt: new Date(submittedAt + 'T00:00:00'), variantId, url: url || undefined, notes: notes || undefined })`. Then call `onSaved()` callback prop to refresh the list and close the form.
- Props: `onSaved: () => void`, `onCancel: () => void`
- Style: Dark zinc aesthetic matching existing forms. Use `bg-zinc-800`, `border-zinc-700`, `text-zinc-100` input styling consistent with existing add forms.

**SubmissionsTab.tsx** — tab root component:
- State: `submissions: Submission[]`, `showAddForm: boolean`, `viewingSnapshot: { data: SubmissionSnapshot, variantName: string | null } | null`
- On mount: call `loadSubmissions()` which does `window.api.submissions.list().then(setSubmissions)`.
- Header area: title "Submissions" + "+ Log Submission" button (toggles `showAddForm`). Follow the same layout as ExperienceTab header.
- When `showAddForm` is true, render `<SubmissionAddForm onSaved={() => { setShowAddForm(false); loadSubmissions(); }} onCancel={() => setShowAddForm(false)} />`.
- Table with columns: Company, Role, Date, Variant, Actions.
  - Use a `<table>` with `<thead>` and `<tbody>`.
  - Table styling: `w-full text-sm text-left`, header cells `text-zinc-500 text-xs uppercase tracking-wider`, body cells `text-zinc-300 border-b border-zinc-800`.
  - Company cell: `<InlineEdit value={sub.company} onSave={(v) => handleUpdate(sub.id, { company: v })} />`
  - Role cell: `<InlineEdit value={sub.role} onSave={(v) => handleUpdate(sub.id, { role: v })} />`
  - Date cell: Display `new Date(sub.submittedAt).toLocaleDateString()`. For editing, use an `<InlineEdit>` that shows the formatted date, and on save parses back. OR simpler: display only (not inline-editable) since date editing via text is error-prone. Use display-only for date.
  - Variant cell: Display `sub.variantName ?? ''` (blank if variant deleted, per locked decision).
  - Actions cell: "View Resume" button + delete button (red X icon or text, immediate delete with no confirmation per locked decision).
- `handleUpdate(id, data)`: Call `window.api.submissions.update(id, data)` then `loadSubmissions()`.
- `handleDelete(id)`: Call `window.api.submissions.delete(id)` then `loadSubmissions()`.
- "View Resume" button: Parse `JSON.parse(sub.resumeSnapshot) as SubmissionSnapshot`, set `viewingSnapshot` state. This opens the SnapshotViewer modal (Task 2).
- When `viewingSnapshot` is not null, render `<SnapshotViewer snapshot={viewingSnapshot.data} onClose={() => setViewingSnapshot(null)} />`.
- Empty state: When `submissions.length === 0` and `!showAddForm`, show centered text: "No submissions yet. Click '+ Log Submission' to log your first application."
- URL and Notes: Show URL as a clickable link icon or small link text in the row if present. Notes can be shown in a tooltip or expandable area — keep it minimal. An acceptable approach: show a small "notes" indicator icon, and let notes be editable via InlineEdit with `multiline={true}` in an expandable row detail, OR just show URL and Notes as additional columns. Use your judgment for clean presentation — the table should remain scannable.

Create `SnapshotViewer.tsx` — a modal overlay that renders a frozen resume snapshot.

**Props:**
```typescript
interface SnapshotViewerProps {
  snapshot: SubmissionSnapshot    // parsed from JSON, contains { layoutTemplate, jobs, skills }
  onClose: () => void
}
```

**Implementation:**
1. Render a modal overlay: fixed position, full viewport, semi-transparent black backdrop (`bg-black/60`), centered content panel (`bg-zinc-900 rounded-lg border border-zinc-700`), max-width ~3xl, max-height ~90vh with overflow-y-auto.
2. Modal header: "Resume Snapshot" title + close button (X).
3. Modal body: Render the resume content using the SAME layout sub-component logic as `VariantPreview.tsx`.

   **CRITICAL: Do NOT import or use VariantPreview directly** — it fetches live data. Instead, replicate the layout rendering inline:
   - Filter `snapshot.jobs` to get `includedJobs` (where `!job.excluded`).
   - Filter `snapshot.skills` to get `includedSkills` (where `!skill.excluded`).
   - Group skills by first tag (same `reduce` logic as VariantPreview).
   - Dispatch to layout based on `snapshot.layoutTemplate`:
     - `'modern'` → two-column layout (skills sidebar + jobs main)
     - `'compact'` → compact single-column
     - default/`'traditional'` → standard single-column

   Copy the `TraditionalLayout`, `ModernLayout`, and `CompactLayout` sub-components from `VariantPreview.tsx` into this file. They are pure presentational components that accept `{ includedJobs, skillGroups }` — they work identically with static data. This is intentional duplication to keep SnapshotViewer fully self-contained and avoid coupling to VariantPreview's data-fetching lifecycle.

4. Close on backdrop click (click handler on overlay div, `e.target === e.currentTarget` check) and on Escape key (useEffect with keydown listener).
5. Close button in top-right corner of modal panel.
