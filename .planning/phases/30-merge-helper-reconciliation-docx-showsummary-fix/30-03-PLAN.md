**Step 2 — Gate the summary paragraph (docxBuilder.ts:122-130):**

Locate the summary block. The current code is:
```typescript
// Professional summary — rendered when profile has summary content
...(profileRow?.summary ? [
  new Paragraph({
    children: [
      new TextRun({ text: profileRow.summary, size: 21, font: fontName }),
    ],
    spacing: { before: 120, after: 200 },
  }),
] : []),
```

Update to:
```typescript
// Professional summary — rendered when showSummary=true AND profile has summary content.
// When showSummary=false, the entire array collapses to [] — both the paragraph AND
// its spacing are suppressed, so the next section (WORK EXPERIENCE) owns its top gap. (Phase 30 D-09)
...(showSummary && profileRow?.summary ? [
  new Paragraph({
    children: [
      new TextRun({ text: profileRow.summary, size: 21, font: fontName }),
    ],
    spacing: { before: 120, after: 200 },
  }),
] : []),
```

**Landmine (D-09 — "no double-gap"):** When `showSummary=false`, the entire spread `...(showSummary && profileRow?.summary ? The next paragraph (WORK EXPERIENCE at ~line 148) has its own `spacing: { before: 240, after: 120 }` which provides the appropriate gap from the contact line above. Do not add any new spacing logic to "compensate" — the React templates use the identical pattern with no compensation needed.
