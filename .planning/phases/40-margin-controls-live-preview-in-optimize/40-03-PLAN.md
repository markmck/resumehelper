  <name>Task 1: Verify margin sliders, live preview, persistence, and revert in the Optimize screen</name>
  <files>(none — verification only)</files>
  <what-built>Three margin sliders (Top/Bottom/Sides) in a collapsible MARGINS section in the Optimize screen, seeded from the analysis's effective margins, with a live paginated preview that updates as you drag, a ~300ms debounced persist via the Phase 39 override path, and an always-rendered "Revert to variant margins" control.</what-built>
  <action>
    Pause for human verification. Do not modify any files. Present the verification steps below, run the app if not already running, and wait for the user's sign-off or defect report. If defects are reported, surface them for a gap-closure plan rather than attempting silent fixes here.
  </action>
  <how-to-verify>
    1. Run the app (`npm run dev`) and open an analysis in the Optimize screen — ideally one whose variant has some excluded bullets and/or accepted rewrites.
    2. Confirm the MARGINS section appears above the Live Preview pane, expanded, with three sliders seeded to sensible inch values (the analysis's effective margins) — LAYOUT-01 / SC#1.
    3. Drag the Top slider down toward 0.4". Expected: the preview margins tighten live as you drag; the readout turns amber once below 0.5"; the value cannot go below 0.4" — LAYOUT-05 / SC#2.
    4. Stop dragging, wait ~1s, then re-open the same analysis. Expected: the margins you set persist (the override was written via the Phase 39 path; no save button was needed) — SC#2.
    5. Confirm the preview shows the analysis's MERGED content — any re-included excluded bullets and accepted rewrites are visible, so overflow from added content is apparent by scrolling — LAYOUT-05 / SC#3.
    6. Confirm "Revert to variant margins" is now enabled (underlined, full opacity). Click it. Expected: sliders re-seed to the variant's margins, the preview reloads at those margins, and the revert control becomes disabled (dimmed, no underline) — LAYOUT-08 / SC#4.
    7. Collapse the MARGINS section via the header. Expected: chevron flips to ▶ and the inline summary shows the three current values like `0.75" / 0.75" / 0.75"`.
  </how-to-verify>
  <verify>
    <human-check>User confirms steps 1-7 pass against the running app, or files specific defects.</human-check>
