## Component Inventory

### 1. Stale Analysis Banner

**Where:** AnalysisResults — renders between the metadata bar and the 4-metric card grid when `isStale === true`.

**Structure:**
```
[warning icon] Analysis may be outdated — resume content changed since this analysis ran.  [Re-analyze →]
```

**Spec:**
- Container: `backgroundColor: var(--color-warning-bg)`, `border: 1px solid rgba(245, 158, 11, 0.25)`, `borderRadius: var(--radius-md)`, padding `8px var(--space-4)`, `marginBottom: var(--space-5)`
- Layout: `display: flex`, `alignItems: center`, `gap: var(--space-3)`
- Warning icon: Unicode `⚠` or inline SVG, color `var(--color-warning)`, 14px
- Body text: 13px, weight 400, color `var(--color-warning)`, `flex: 1`
- "Re-analyze" button: ghost link style — `background: none`, `border: none`, `color: var(--color-warning)`, 13px, weight 600, cursor pointer, `padding: 0`, `marginLeft: auto`
- Does NOT block any other action (non-modal, non-disabling)

### 2. Inline Edit Fields — Company and Role

**Where:** AnalysisResults metadata bar — replaces the static `<p>` for role (18px/600) and company (13px/secondary color).

**Display mode (not editing):**
- Role: 18px, weight 600, color `var(--color-text-primary)`, cursor `text` on hover with a subtle underline-dotted hint (`borderBottom: '1px dashed var(--color-border-emphasis)'`)
- Company: 13px, weight 400, color `var(--color-text-secondary)`, same hover cursor hint

**Edit mode (editing):**
- `<input>` element, `autoFocus`, `width: auto` (sized to content using `size` attribute or `min-width`)
- Role input: 18px, weight 600, color `var(--color-text-primary)`, `backgroundColor: var(--color-bg-input)`, `border: 1px solid var(--color-accent)`, `borderRadius: var(--radius-sm)`, padding `4px 8px`, `outline: none`
- Company input: 13px, weight 400, color `var(--color-text-secondary)`, same bg/border treatment but smaller, padding `4px 8px`
- Commits on `blur` or `Enter` keydown — calls `window.api.jobPostings.update()`
- Cancels on `Escape` — reverts to original value without IPC call
- On successful persist: updates local state only, no full re-fetch (Pitfall 3 avoidance)

### 3. Log Submission Button — OptimizeVariant

**Where:** OptimizeVariant action bar (currently only contains "Back" navigation).

**Spec (matches AnalysisResults existing ghost variant):**
- `padding: '8px 16px'`
- `backgroundColor: 'var(--color-bg-surface)'`
- `color: 'var(--color-accent)'`
- `border: '1px solid var(--color-accent)'`
- `borderRadius: 'var(--radius-md)'`
- `fontSize: 'var(--font-size-sm)'`
- `fontFamily: 'var(--font-sans)'`
- `cursor: 'pointer'`

**Behavior:** Calls `onLogSubmission?.(analysisId)` — navigates to SubmissionLogForm with pre-fill. No inline form or modal (D-02).

### 4. Orphaned Override Row

**Where:** OptimizeVariant suggestion list — renders in place of a normal suggestion row when `override.isOrphaned === true`.

**Spec:**
- Row background: `var(--color-bg-raised)` (same as normal rows)
- Override text: 13px, weight 400, color `var(--color-text-muted)` (`#4a4a52`), `textDecoration: 'line-through'`
- Notice badge: inline span, `backgroundColor: var(--color-danger-bg)`, `color: var(--color-danger)`, 11px, weight 600, `borderRadius: var(--radius-sm)`, padding `4px 8px`
- Notice text: "Original bullet was deleted."
- No action buttons on orphaned rows (accept/dismiss buttons hidden)

### 5. Auto-Extract Feedback — NewAnalysisForm

**Where:** NewAnalysisForm company and role fields — shown transiently when regex extraction fires on paste.

**Spec:**
- No visual indicator required beyond the fields auto-filling
- Fields use existing input styling from NewAnalysisForm
- Company/role inputs: placeholder text changes to reflect auto-fill state via React controlled value only
- No toast, no badge — silent fill is sufficient (user sees the values appear)

## Interaction States

### Inline Edit Lifecycle

| State | Visual |
|-------|--------|
| Default (display) | Static text, cursor `text` on hover, dotted bottom border on hover |
| Editing | Input with accent border, autoFocus, value controlled |
| Saving (pending IPC) | No loading indicator — IPC is fast enough (local SQLite) |
| Saved | Reverts to display mode with updated value |
| Error | Reverts to original value, no toast needed (silent — local DB failure is rare) |
| Cancelled (Escape) | Reverts to original value without any IPC call |

### Stale Banner Lifecycle

| State | Visual |
|-------|--------|
| Not stale | Banner not rendered |
| Stale | Amber banner rendered below metadata bar, above metric cards |
| Re-analyze clicked | Calls `onReanalyze(raw.jobPostingId, raw.variantId)` — same as existing "Re-analyze" button in AnalysisResults |

| Primary CTA (both screens) | "Log Submission" |
| Inline edit role placeholder | "Role title" |
| Inline edit company placeholder | "Company name" |

| Stale banner action | "Re-analyze" |
| Orphaned override notice | "Original bullet was deleted." |
| Empty stale state (not stale) | (no element rendered — banner is not shown) |
| Error state (load failure) | "Failed to load analysis" + `{error message}` + "Back to analyses" button — existing copy, unchanged |
