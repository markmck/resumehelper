### 1. `SkillChipGrid` (new — replaces `SkillList`)

Top-level container. Hosts `DndContext`. Renders the panel with section header, all `CategoryBlock` items, `DropZoneNewCategory`, "Add category" button, and hint text.

```
.panel
  .section-head
```

**Panel styles:**
- `background: var(--color-bg-base)`

### 3. `CategoryNameEditor`

Inline editable category name. Toggles between read (span) and edit (input) mode.

**Read mode:**
- `<span class="category-name">`
- `font-size: 11px` (`--font-size-xs`)
- `font-weight: 500`
- `text-transform: uppercase`
- `letter-spacing: 0.05em`
- `color: var(--color-text-tertiary)` (#6b6b74)
- Clicking the span OR pressing "Rename" button enters edit mode

**Edit mode:**
- `<input class="category-name-input" autoFocus />`
- Same font/size/weight/transform as read mode
- `background: transparent; border: none; outline: none`
- `width: 180px`

- Revert on: `Escape` key

### Drag Flow (full sequence)

1. User presses pointer on a chip → `useDraggable` activates, `isDragging = true`
2. Source chip: `opacity: 0.5`, `border-color: var(--color-accent)`
3. `DragOverlay` appears: floating chip clone at full opacity with accent border
4. Chip enters a category zone: that zone's `.chip-grid` gets `.drop-target` class
5. User releases over a different category: `onDragEnd` fires
   - `active.id` = skill integer id
   - `over.id` = `"category-${cat.id}"` string
   - Parse category id: `parseInt(over.id.replace('category-', ''))`
   - Optimistic state update: mutate `categoryId` in local skills array immediately
   - IPC call: `window.api.skills.update(skillId, { categoryId: targetCategoryId })`
6. User releases over `"drop-zone-new-category"`: trigger new-category creation flow (see Copywriting)
7. User releases over nothing (`over = null`): no action, chip returns to origin

### Category Delete Flow

1. User clicks "Delete" on a category
2. Inline confirmation replaces action buttons: "Delete — this will move N skills to Uncategorized. Confirm?"
3. Confirm → `skills:categories:delete(catId)` — FK `ON DELETE SET NULL` moves all skills to `categoryId = null`
4. Those skills appear in the "Uncategorized" group
5. Keep Category → action buttons restored, no change

### Uncategorized Group

Skills with `categoryId = null` render in a special "Uncategorized" category block at the bottom of the list (above the drop zone). This block:
- Has no delete button (cannot delete Uncategorized)
- Has no drag handle (non-functional like all handles in this phase)
- Category name: "UNCATEGORIZED" (uppercase, same `.category-name` style)
- Has "+ Add" chip and can receive drops from other categories
- Only visible if at least one skill has `categoryId = null`; hidden if no uncategorized skills exist
